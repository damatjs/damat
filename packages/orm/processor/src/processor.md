# ORM Processor Module

The `@damatjs/orm-processor` package provides internal tooling for the ORM to manage database schemas. It is responsible for analyzing schema differences, generating SQL for migrations, and managing schema snapshots.

## Directory Structure

```
processor/
├── diff/            # Schema differentiation engine
├── snapshot/        # Schema snapshot generation and storage
├── sqlGenerator/    # SQL query generation from schema differences
├── types/           # Type definitions for snapshots and diffs
├── index.ts         # Main entry point
└── processor.md     # This documentation
```

## Key Features

### 1. Schema Diffing (`diff/`)
Compares two database schemas (e.g., current code vs. last snapshot) to detect changes. It effectively identifies:
- Added, modified, or removed tables.
- Added, modified, or removed columns.
- Alterations to indexes and foreign keys.

This is the engine that drives automatic migration generation.

### 2. Snapshots (`snapshot/`)
Generates and manages structured JSON representations of the database schema at a specific point in time. 
Snapshots enable reliable migration generation without needing a live database connection, as we can simply compare the current schema defined in code against the last saved snapshot.

### 3. SQL Generation (`sqlGenerator/`)
Takes the abstract differences generated by the diff engine and produces dialect-specific SQL statements (e.g., PostgreSQL) to migrate the database schema. It knows how to translate things like "Column was added" into `ALTER TABLE ... ADD COLUMN ...`.

## Internal Usage

This module is primarily used internally by `@damatjs/orm-migration`, the CLI tools, or connection managers to facilitate schema migrations. It does not provide end-user APIs for defining models or executing database queries; those responsibilities are handled by `@damatjs/orm-model`.
