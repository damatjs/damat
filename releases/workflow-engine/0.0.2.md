# @damatjs/workflow-engine 0.0.2

> First pre-alpha release: the saga-style, in-process workflow engine on Effect-TS — typed steps with compensation, retry/timeout, and Redis-backed distributed locking.

## What changed

`0.0.2` is the first published (pre-alpha) release of `@damatjs/workflow-engine`.
It provides the orchestration primitive used across Damat: you author each unit
of work as a plain `async` step with an optional compensation (undo) function,
compose steps inside an Effect generator, and the engine handles retries,
timeouts, reverse-order rollback on failure, and optional mutual exclusion via a
Redis lock. State lives in memory for the duration of a run — there is no
persisted journal and no resume-after-crash.

At this baseline the package offers:

- **`createStep(name, invoke, compensate?, config?)`** — build a typed
  `StepDefinition<I, O>`. `invoke(input, ctx)` does the work; `compensate(input, output, ctx)`
  is registered as a scoped finalizer that runs only if a _later_ step fails.
- **`createWorkflow(name, definition, config?)`** — build a
  `WorkflowDefinition<I, O>` exposing `execute` and `executeWithLock`, each
  resolving to a `WorkflowResult<O>` discriminated union (`{ success: true, ... }`
  / `{ success: false, error, ... }`) rather than throwing for ordinary step
  failures.
- **`executeStep` / `runStep`** — run a step inside the workflow generator,
  applying retry, timeout, and compensation registration.
- **Control-flow helpers** — `skipStep`, `parallel`, `when`, `ifElse`.
- **Retry & timeout** — `RetryPolicy` with exponential backoff and the
  `RetryPolicies` presets (`none`, `once`, `standard`, `aggressive`, `patient`);
  default no-retry, a 30s step timeout, and a 5min workflow timeout.
- **Saga compensation** — successful steps with a `compensate` fn add finalizers
  that fire in reverse order when the workflow scope closes with a failure.
- **Distributed locking** — `acquireWorkflowLock` / `releaseWorkflowLock` /
  `extendWorkflowLock` / `isWorkflowLocked`, backed by `@damatjs/redis`, plus
  `executeWithLock` for "only one run per business id at a time".
- **Typed error hierarchy** — `WorkflowError` and subclasses
  (`StepExecutionError`, `StepTimeoutError`, `MaxRetriesExceededError`,
  `CompensationError`, `WorkflowLockError`), each carrying a programmatic `code`.
- **Structured logging** via `@damatjs/logger` (no-op when no global logger is
  set), and `Effect` / `Scope` re-exported from `effect` for convenience.

## Added

- The full step / workflow / control-flow / retry / locking / error surface
  described above.

## Action required

None — drop-in. New consumers add the dependency and import from the package root:

```bash
bun add @damatjs/workflow-engine
```

This is a pre-alpha release: APIs and behavior may still change in later versions
(see [0.1.0](./0.1.0.md) for the stabilization minor).
