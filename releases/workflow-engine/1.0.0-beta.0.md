# @damatjs/workflow-engine 1.0.0-beta.0

> Adds a non-authoritative observation and cancellation seam for durable pipeline integration.

## What changed

Workflow execution can report identity-only lifecycle events without turning
the in-process saga engine into a persistence system. Pipelines use the observer
to associate a durable workflow node with its internal step activity.

## Added

- `WorkflowExecutionOptions` with `executionId`, outer `signal`, and `observer`.
- Workflow, step-attempt, and compensation start/success/failure events.

## Changed / improved

- `execute` and `executeWithLock` accept additive execution-options arguments.
- Locked and unlocked execution both forward cancellation and observation.
- Observer failures are logged and never replace workflow outcomes.

## Breaking

- None.

## Action required

None. Pass execution options only when tracing or an outer runtime must cancel a workflow.
