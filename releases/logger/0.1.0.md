# @damatjs/logger 0.1.0

> First published minor release: the stabilized, zero-dependency structured logger — ten levels, three formats, child/prefixed loggers, an opt-in dual-format file transport, a no-op logger, and a process-global singleton.

## What changed

`0.1.0` is the baseline minor release of `@damatjs/logger`. It is the logging
layer the rest of the Damat stack depends on, built only on Node's `fs`/`path`
and raw ANSI escapes — no runtime dependencies. At this baseline the package
provides:

- **`Logger`** (`new Logger(config)` / `Logger.create(config)`) with ten leveled
  methods — `debug`, `info`, `progress`, `waiting`, `cached`, `success`, `warn`,
  `error`, `fatal`, `skip` — plus `child`, `withPrefix`, `request`, and `close`.
  Each level maps to a numeric threshold (`debug` = 0 … `skip` = 9); an entry is
  emitted when its threshold is `>=` the configured `level` (default `"info"`).
- **Three output formats** selected by `LoggerConfig.format`: `json` (the raw
  entry serialized), `pretty` (colorized badge + label + message + context +
  error block), and `simple` (which takes the `pretty` code path). Color is
  auto-detected and honors `NO_COLOR` / `FORCE_COLOR` / `TERM=dumb` / TTY.
- **`ChildLogger`** via `child(context)` and `withPrefix(name)` — lightweight,
  context-merging, prefix-nesting (`a:b`) loggers that delegate formatting and
  I/O to the parent `Logger`.
- **`NoopLogger` + `NOOP_LOGGER`** — an `ILogger` that discards everything while
  still supporting `child`/`withPrefix` chaining, for tests and silent libraries.
- **`FileTransport`** — an opt-in, buffered, dual-format file sink. When enabled
  (via `file.enabled: true` or the `LOG_FILE` / `LOGGING_FILE_ON` env vars) it
  writes date-stamped `logs/YYYY-MM-DD_all.log` and `.md` files, flushes on a
  timer (default 1s) or on `close()`, and rotates a file past `maxSizeBytes`
  (default 10 MiB). It is completely inert when not enabled.
- **A process-global singleton** (`createLogger`, `setGlobalLogger`, `getLogger`,
  `clearGlobalLogger`, `closeLogger`, `isLoggerConfigured`, `createContextLogger`)
  plus top-level convenience wrappers (`debug`/`info`/`progress`/`cached`/
  `waiting`/`warn`/`error`/`fatal`) that call the matching method on
  `getLogger()`.
- **Banner utilities** (`separator`, `successBanner`, `errorBanner`) for
  standalone console rules and one-line status banners.

## Added
- The full public surface above, re-exported from the package root
  (`types`, `logger`, `child`, `noop`, `file-transport`, `global`, `utils`).
  `colorizer`, `colors`, and `formatter` stay internal.

## Changed / improved
- **Child-logger prefix routing now reaches the formatter.** Leveled methods
  funnel through a shared `Logger.logWithPrefix(level, message, prefix, context?,
  error?)` entry point. A `Logger`'s own methods call it through the private
  `log()` wrapper with the logger's configured `prefix`; a `ChildLogger` calls it
  directly with the **child's** prefix. Before this, `ChildLogger` forwarded to
  the parent's level methods (`parent.info(...)`, etc.), which formatted with the
  *parent's* prefix — so a prefix added via `ChildLogger.withPrefix()` was dropped
  from the output. After this change, a child's prefix appears in the rendered
  line.

  ```ts
  // before: child prefix was lost
  this.parent.info(message, this.merge(context));

  // after: child prefix is forwarded to the formatter
  this.parent.logWithPrefix("info", message, this.prefix, this.merge(context));
  ```

## Breaking
- None.

## Action required

None — drop-in. New consumers add the dependency and import from the package
root:

```bash
bun add @damatjs/logger
```

Inside the monorepo, depend on it as a workspace package with `"@damatjs/logger": "*"`.
