# @damatjs/cli 0.1.4

> Adds a `defaultCommand` to the CLI framework, so a CLI can run its primary
> command without the user typing its name.

## What changed

`runCli` previously required the first argument to name a registered command;
anything else printed "Unknown command" and the help. A CLI config can now set
`defaultCommand`, and when the first argument is not a known command the runner
falls back to that command, passing the **whole** argument list to it as
positionals.

```ts
runCli({
  name: "create-damat-app",
  commands: [createCommand],
  defaultCommand: "create",   // ← new
});
```

```bash
# Before: required the command name
create-damat-app create my-app
# After: the default command runs implicitly
create-damat-app my-app          # == create-damat-app create my-app
```

This is what lets `create-damat-app <name>` work without typing `create`.

## Added

- `CliConfig.defaultCommand?: string` — fallback command used when the first
  argument is not a registered command name.

## Changed / improved

- `runCli` resolves an unknown leading argument to `defaultCommand` (when set)
  instead of erroring; it still prints "Unknown command" + help when there is no
  default.

## Breaking

- None. Without `defaultCommand` the behavior is unchanged.

## Action required

- None — opt in by setting `defaultCommand` on your CLI config.
