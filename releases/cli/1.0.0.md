# @damatjs/cli 1.0.0

> The core CLI becomes a framework-neutral, embeddable runtime with injected process boundaries and invocation-local state.

## What changed

The package previously constructed a Damat logger, read process state throughout
the run loop, shared registry/config state at module scope, printed presentation
through global console calls, and terminated the process from command paths.

It now accepts a `CliDefinition` and partial `CliRuntime`, creates isolated
registry and optional project-config state for each call, routes presentation
through structural output contracts, and returns `CliRunResult`. The Damat
executable owns its logger, presentation defaults, environment policy, and
process exit code.

Before:

```ts
runCli(config).catch((error) => {
  reportError(new Logger({ timestamp: false }), error, {
    prefix: "Fatal error",
  });
  process.exit(getExitCode(error));
});
```

After:

```ts
const runtime = createDamatRuntime();

runCli(definition, runtime)
  .then((result) => {
    process.exitCode = result.exitCode;
  })
  .catch((error) => {
    reportError(runtime.logger, runtime.output, error, {
      prefix: "Fatal error",
      verbose: runtime.args.includes("--verbose"),
    });
    process.exitCode = getExitCode(error);
  });
```

## Added

- `CliRuntime`, `CliRunResult`, `CliLogger`, and `CliOutput` contracts.
- `createRuntime` for dependency-free process-backed defaults.
- `createCommandRegistry` for isolated command state.
- Shared `executeCommand` and result-based command routing.
- `CliCapability`, ordered composition, and a standalone package test harness
  under `@damatjs/cli/testing`.
- Repository line-limit checker and targeted CLI command.

## Changed / improved

- Default/subcommand and CAC leaf routes share validation, config, error, and
  handler execution.
- Config caching belongs to a `withConfig(loader, cwd)` accessor.
- Project configuration is optional; omitting `configLoader` creates no accessor.
- Banner and verbose presentation are opt-in.
- Error verbosity is explicit and contains no Damat environment lookup.
- Core dependencies are reduced to `cac`.

## Breaking

- `runCli` returns `Promise<CliRunResult>` and does not terminate the process.
- `CliConfig` is renamed to `CliDefinition` to distinguish the executable
  definition from optional project configuration.
- Registry singleton helpers are replaced by `createCommandRegistry`.
- `loadConfig` requires `cwd`; caching moves to `withConfig`.
- Help, banner, and output writers require `CliOutput`.
- Core no longer provides a Damat logger automatically.
- Omitted banner and verbose config no longer enable presentation.

## Action required

1. Await `runCli` and apply its `exitCode` in the executable boundary.
2. Inject a runtime when custom args, cwd, env, logging, or output are required.
3. Replace registry singleton calls with an owned registry.
4. Replace process-wide config-cache calls with `withConfig(loader, cwd)`.
5. Pass `CliOutput` to help/banner/output helpers.
6. Configure `banner` and `verbose.enabled` explicitly when desired.
7. Rename `CliConfig` references to `CliDefinition`.
