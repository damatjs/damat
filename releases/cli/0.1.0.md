# @damatjs/cli 0.1.0

> First minor release: the stabilized declarative CLI framework that turns a single `CliConfig` object into a runnable command-line tool.

## What changed

This is the package's first-of-its-kind release. `@damatjs/cli` is the general
CLI **framework** at the base of Damat's command-line tools — the user-facing
`@damatjs/damat-cli` (`damat`) and `@damatjs/orm-cli` (migrations) are both built
on it. You hand `runCli` a declarative `CliConfig` describing commands, options,
and handlers; it builds a [cac](https://github.com/cacjs/cac) instance, mirrors
your commands into a singleton registry, parses `process.argv`, validates and
coerces options, dispatches to your handler with a typed `CommandContext`,
optionally loads a project config file, and renders help and a banner. It owns
the process lifecycle (it calls `process.exit`).

Earlier `0.0.x` tags were pre-alpha build-and-packaging iterations; `0.1.0` is
the first version stabilized for use, with expanded test coverage and known
issues resolved.

What the package provides at this baseline:

- **One-call entry point** — `runCli(config)` runs the full loop: validate
  `name`/`version`, reset the registry, build cac, register the optional
  `--verbose` global flag, register commands, add a `help [command]` command,
  mirror leaf commands into cac, print the banner, then dispatch.
- **Declarative command model** — `CliConfig`, `Command`, `CommandOption`,
  `CommandContext`, `CommandResult`, `BannerConfig`, `VerboseConfig`,
  `ConfigLoader`, `ParsedCommand`, plus `HelpTemplateFn` / `ErrorHandlerFn`.
  Commands support `aliases`, `usage`, `examples`, nested `subcommands`, and a
  typed `handler(ctx) => Promise<{ exitCode }>`.
- **Singleton command registry** — `getRegistry`, `registerCommand`,
  `getCommand`, `getAllCommands`, `clearRegistry`. Subcommands are namespaced as
  `parent:child`, aliases share the name map, and `getAll()` dedupes by command
  `name`.
- **Option validation & coercion** — `coerceOptions` / `coerceOptionValue`
  (`string`/`number`/`boolean`), `applyDefaults`, and `validateOptions` (throws
  `MissingRequiredOptionError`), run automatically on the leaf-command path.
- **Project config loading** — `loadConfig` (cached process-wide),
  `clearConfigCache`, and the `withConfig` closure helper; a non-null result is
  injected as `ctx.options.config` on the leaf path.
- **Help & output** — `printDefaultHelp`, `printCommandSpecificHelp`,
  `formatCommandLine` / `formatOptionLine` / `formatCommandHelp`, `printBanner`
  (boxed/minimal/none), and logger-backed `printError` / `printSuccess` /
  `printInfo` / `printSection`.
- **Error hierarchy** — `CliError` (base, `exitCode` default `1`) plus
  `CommandNotFoundError`, `MissingRequiredOptionError`, `ConfigLoadError`, and
  `CommandRegistrationError`.

## Added

- `runCli` and the run loop (arg parsing via `parseCommandArgs`,
  `buildCommandContext`, `resolveCommandName`, `registerSingleCommand`,
  `handleHelpCommand`, `buildOptionFlag`).
- The command registry singleton and helpers.
- The `utils/validate` pipeline (`coerceOptions`, `applyDefaults`,
  `validateOptions`) and the `utils/output` helpers.
- Project-config loading (`loadConfig`, `clearConfigCache`, `withConfig`).
- Help rendering, banner rendering, and the `CliError` hierarchy.
- The full type surface (`CliConfig`, `Command`, `CommandOption`,
  `CommandContext`, `CommandResult`, `CommandRegistry`, `ParsedCommand`,
  `BannerConfig`), all re-exported from the package root.

## Changed / improved

- Core functionality stabilized and test coverage expanded relative to the
  `0.0.x` pre-alpha line.

## Breaking

- None — this is the baseline minor release.

## Action required

None — drop-in for new projects. Add the dependency and call `runCli` from your
entry point:

```bash
bun add @damatjs/cli
```

```ts
import { runCli } from "@damatjs/cli";

runCli({
  name: "my-cli",
  version: "1.0.0",
  commands: [/* ... */],
});
```
