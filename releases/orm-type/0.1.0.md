# @damatjs/orm-type 0.1.0

> First published minor release: the shared, dependency-free TypeScript type vocabulary that every other Damat ORM package speaks, stabilized for general use.

## What changed

`0.1.0` is the baseline minor release of `@damatjs/orm-type`. It sits at the very
bottom of the ORM stack: it ships **only types** — no runtime code, no classes,
no functions — so it can be imported anywhere (model DSL, query builder,
migration engine, CLI, codegen) without pulling in implementation. It has a
single entry point (`.`) and re-exports the `pg` driver types from
`@damatjs/deps` plus four type groups.

At this baseline the package provides:

- **Re-exported driver types** — `Pool`, `PoolClient`, `QueryResultRow` from
  `@damatjs/deps/pg`.
- **Connection types** (`connection/`) — `DbConnection` (the driver abstraction:
  `pool`, `query`, `transaction`, `getClient`, `close`, `isConnected`,
  `getStats`), `DbPoolConfig`, `DbPoolConfigWithExtras`, `DbConnectionConfig`,
  `ConnectionStatus`, `PoolStats`, `TransactionOptions`,
  `TransactionIsolationLevel`, `EntityConstructor<T>`, `QueryContext`.
- **Model / schema snapshot types** (`model/`) — `ColumnType` (the union of ~80
  exact PostgreSQL SQL type names), `ColumnSchema`, `TableSchema`, `ModuleSchema`,
  `EnumSchema`, the index types (`IndexType`, `IndexColumn`, `IndexSchema`), the
  constraint union (`ConstraintType`, `UniqueConstraint`, `PrimaryKeyConstraint`,
  `CheckConstraint`, `ExcludeConstraint`, `ConstraintSchema`), the foreign-key
  types (`ForeignKeyAction`, `ForeignKeyType`, `ForeignKeySchema`,
  `ForeignKeySchemaMatch`), and the relation types (`RelationType`,
  `RelationSchema`, and the builder option payloads `RelationOptions`,
  `LinkConfig`, `ConstraintOptions`).
- **Query types** (`query/`) — `WhereOperators`, `WhereClause<Cols>`,
  `WhereConditionValue`, `RawWhereClause`, `OrderByClause`, `OrderDirection`,
  `BuiltQuery`, the five JSON descriptors (`SelectDescriptor`, `InsertDescriptor`,
  `UpdateDescriptor`, `DeleteDescriptor`, `UpsertDescriptor`) and their
  `QueryDescriptor` union, `RelationDescriptor`, and the serializable clause
  counterparts `WhereConditionJson` / `OrderByJson`.
- **Migration types** (`migration/`) — `OrmModule` (`{ id, name, path, resolve }`
  module manifest entry) and `OrmModuleContainer` (keyed module map).

## Added
- The full connection / model / query / migration type surface described above,
  all re-exported from the package root, plus the re-exported `pg` driver types.

## Changed / improved
- None — this is the baseline release.

## Breaking
- None.

## Action required

None — drop-in. Consumers add the dependency and import types from the package
root:

```bash
bun add @damatjs/orm-type
```

Most application code never imports it directly — it gets these types
transitively through `@damatjs/orm-model` (which does
`export * from "@damatjs/orm-type"`). Import `@damatjs/orm-type` directly only in
low-level ORM packages.
