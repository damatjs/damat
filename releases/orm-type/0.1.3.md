# @damatjs/orm-type 0.1.3

> `OrmModule` gains an optional `kind?: "module" | "link"` field so the toolchain can tell a cross-module link directory apart from an ordinary module, as part of the cross-module links work.

## What changed

The cross-module links feature (`@damatjs/link`) needed the migration/CLI layer
to distinguish a junction-table **link** directory from an ordinary module while
both flow through the same registration and migration pipelines. The shared
manifest type that describes a registered module — `OrmModule` in
`migration/index.ts` — got one new optional field to carry that distinction.

Before, an `OrmModule` was four fields:

```ts
export interface OrmModule {
  id: string;
  name: string;
  path: string;
  resolve: string;
}
```

After, it carries an optional `kind` tag:

```ts
export interface OrmModule {
  id: string;
  name: string;
  path: string;
  resolve: string;
  /** "link" for a cross-module link directory; undefined for ordinary modules. */
  kind?: "module" | "link";
}
```

The link resolver in `@damatjs/link` sets `kind: "link"` on the per-owner
`link:<owner>` migration modules it emits; ordinary modules leave `kind`
`undefined`, so existing producers and consumers of `OrmModule` keep compiling
unchanged. `OrmModuleContainer` (the keyed `{ [key]: OrmModule }` map) is
unchanged.

This is a types-only change in a types-only package — there is no runtime
behavior here. The field exists so the migration engine, the `damat-orm` CLI, and
codegen can branch on `kind` when handling link directories.

## Added

- `OrmModule.kind?: "module" | "link"` — optional tag identifying a cross-module
  link directory (`"link"`) versus an ordinary module (`undefined`).

## Changed / improved

- None beyond the added field.

## Breaking

- None. `kind` is optional, so every existing `OrmModule` value and consumer
  remains valid without modification.

## Action required

None — drop-in upgrade. The new field is optional and additive. Code that
produces `OrmModule` values can set `kind: "link"` for link directories if it
needs the distinction; code that reads `OrmModule` may now branch on `kind`. No
existing call site needs to change.
