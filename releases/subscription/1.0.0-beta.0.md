# @damatjs/subscription 1.0.0-beta.0

`SubscriptionProviderService(config)` now extends `ModuleService` and requires
create/get/list, change, cancellation, pause, resume, and raw-body webhook
verification operations. An ordinary structurally compatible module service
remains valid.

Subscription records now carry quantity, period dates, and period-end
cancellation state. Runtime record guards and a concrete normal-module fixture
exercise the full lifecycle.

Subscription modules own plans, entitlements, credentials, routes, persistence,
and remote idempotency policy. The framework supplies no provider-specific
lifecycle.
