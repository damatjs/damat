# @damatjs/provider 1.0.0

> Providers are ordinary modules whose services may implement a standardized
> role contract.

## Added

- `ProviderService({ role, ...moduleServiceConfig })`, an abstract class factory
  extending `ModuleService` without losing models, accessors, credentials,
  transactions, caching, or events.
- Provider role markers and top-level role-to-module binding types.
- Runtime validation for binding shape and marked role mismatches.

## Removed before 1.0

The rejected source-factory registry, `ProviderContext`, capability discovery,
provider route/middleware descriptors, provider health, and provider shutdown
lifecycle were removed without a compatibility layer. Normal module mechanisms
own installation, initialization, persistence, routes, and lifecycle.
