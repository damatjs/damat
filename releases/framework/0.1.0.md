# @damatjs/framework 0.1.0

> First minor release: the stabilized Damat backend bootstrap that turns a `damat.config.ts` plus a folder of route files into a running HTTP server.

## What changed

This is the package's first-of-its-kind release. `@damatjs/framework` is the entry
point at the top of the Damat backend stack: it loads and validates config, wires
the service layer, builds a Hono app from file-based routes, serves it, and shuts
it down cleanly. Earlier `0.0.x` tags were pre-alpha build-and-packaging
iterations; `0.1.0` is the first version stabilized for use, with expanded test
coverage and known issues resolved.

What the package provides at this baseline:

- **One-call boot** — `start(cwd?)` runs the full pipeline: `loadConfigAsync()`
  dynamically imports `damat.config.ts` and validates `projectConfig` →
  `initializeServices(config)` brings up logger, PostgreSQL pool (via
  `@damatjs/services` `PoolManager`), Redis, and modules → `bootstrap()` builds the
  Hono app → `setupShutdownHandlers()` installs SIGINT/SIGTERM handling →
  `startServer()` serves via `@hono/node-server`. `runEntry()` wraps `start()` with
  top-level error handling and `process.exit(1)`.
- **Typed config** — `defineConfig(config)` for `damat.config.ts`, with
  `AppConfig` / `ProjectConfig` / `HttpConfig` / `HttpRateLimitConfig` /
  `HttpAuthConfig` / `ModuleConfig` / `ServicesConfig`. Config loading is
  async-only (`loadConfigAsync`; `loadConfig` throws by design) and cached
  process-wide (`clearConfigCache()` resets it).
- **File-based router** — `defineRoute(handler)` in `src/api/routes/**/route.ts`
  files, scanned and sorted for specificity (static > `:param` > `*`, shallower
  first), with declarative per-method validation, rate limiting, and auth resolved
  by precedence (method > route > global).
- **Standard middleware & envelope** — secure headers, request timing, request IDs
  with a child logger, CORS, and structured error handling, all producing a
  consistent `{ success, data | error, meta }` response envelope.
- **Built-in routes** — `/health` whenever a health check is provided, plus
  development-only `/damat` info and `/damat/api/routes` introspection.
- **Subpath exports** — import the narrowest piece you need:
  `@damatjs/framework/config`, `/router`, `/bootstrap`, `/middleware`,
  `/handlers`, `/server`, `/shutdown`, `/services`, `/entry`. The barrel also
  re-exports all of `@damatjs/services`.

## Added

- `start` / `runEntry` boot pipeline (`@damatjs/framework/entry`).
- `defineConfig`, `loadConfigAsync`, `clearConfigCache`, and the config types
  (`@damatjs/framework/config`).
- `bootstrap(options)` that builds the Hono app without serving it.
- File router: `createFileRouter`, `defineRoute`, `response` helpers,
  `resolveMethodConfig`, and the scanner (`scanDirectory`, `sortRoutes`,
  `folderToUrlPath`).
- Middleware: `setupMiddleware`, `errorHandler`, `notFoundHandler`,
  `requestSetup`, `createRateLimitMiddleware`, `createAuthMiddleware`,
  `corsConfigSetter`.
- Built-in handlers: `createRootRoute`, `createApiRoutesRoute`,
  `createHealthRoute`.
- `startServer` and shutdown handling (`setupShutdownHandlers`,
  `registerShutdown`).
- Service wiring: `initializeServices`, logger/database/redis helpers, and the
  module registry (`getModule`, `hasModule`, `clearModules`, `getAllModules`,
  `initModules`, `registerModule`).

## Changed / improved

- Core functionality stabilized and test coverage expanded relative to the
  `0.0.x` pre-alpha line.

## Breaking

- None — this is the baseline minor release.

## Action required

None — drop-in for new projects. Define `damat.config.ts` with `defineConfig`,
add route files under `src/api/routes/`, and call `start()` from your entry point.
