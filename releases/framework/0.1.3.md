# @damatjs/framework 0.1.3

> Adds the top-level `links` config field, wires cross-module link directories as a `link` module at boot, and re-exports the `@damatjs/link` authoring surface from the framework barrel.

## What changed

Cross-module links (`@damatjs/link`) now boot through the framework. You declare a
relationship between two models that live in different modules with `defineLink`,
collect them with `collectLinkModels`, and default-export `defineLinkModule(...)`
from a directory (typically `src/links/`). The framework picks that directory up,
registers it as a `link` module, and hydrates it alongside your other modules.

Before, `damat.config.ts` only carried `projectConfig`, `modules`, and `services`,
and the link authoring helpers had to be imported from `@damatjs/link` directly:

```ts
// before — no place to register links; link helpers from a separate package
import { defineConfig } from "@damatjs/framework";
import { defineLink } from "@damatjs/link";

export default defineConfig({
  projectConfig: { /* ... */ },
  modules: { user: { resolve: "./src/modules/user", id: "user" } },
});
```

After, there is a top-level `links` field, and the link authoring surface is
re-exported from the framework so everything comes from one package:

```ts
// after — `links` field + link helpers from @damatjs/framework
import { defineConfig } from "@damatjs/framework";

export default defineConfig({
  projectConfig: { /* ... */ },
  modules: { user: { resolve: "./src/modules/user", id: "user" } },
  links: "./src/links", // string | string[]
});
```

```ts
// src/links/index.ts
import { defineLink, defineLinkModule, collectLinkModels } from "@damatjs/framework";

const userOrganization = defineLink(/* ... */);

export const links = [userOrganization];
export const models = collectLinkModels(links); // for migrate + codegen
export default defineLinkModule(links);         // for boot
```

At boot, `initializeServices` calls `resolveLinkModuleEntries(config.links, cwd)`
(from `@damatjs/link`), appends the resulting `{ id, resolve }` entries to the
module-config list, and initializes them through the existing `initModules` path.
It then calls `setLinkModuleResolver((id) => getModule(id))` so the link service
can hydrate linked rows by calling other modules' services. `getModule("link")`
resolves the link service (`create` / `dismiss` / `fetch` plus a nested `graph`
query). The junction tables migrate and type-generate through the existing
pipelines, and the `damat-orm` CLI picks the same link directories up.

## Added

- Top-level `links` field on `AppConfig` (`string | string[]`) — directory path(s)
  whose `index.ts` default-exports `defineLinkModule(...)` and exports `models`.
- Boot wiring: `resolveLinkModuleEntries` + `setLinkModuleResolver` integrated into
  `initializeServices`, registering link directories as `link` module(s).
- Barrel re-export of the `@damatjs/link` authoring surface (`defineLink`,
  `defineLinkModule`, `collectLinkModels`, and related types) from
  `@damatjs/framework`.
- `@damatjs/link` added as a framework dependency.

## Changed / improved

- `initializeServices` now builds its module list from `config.modules` **plus**
  resolved link entries, so links share the module init path and lifecycle.

## Breaking

- None. The `links` field is optional and opt-in; configs without it behave
  exactly as before, and `setLinkModuleResolver` is a no-op when no link module is
  registered.

## Action required

None unless you want cross-module links. To adopt them:

1. Author your links in a directory (e.g. `src/links/index.ts`): `export const
   links = [...]`, `export const models = collectLinkModels(links)`, and
   `export default defineLinkModule(links)`. Import the helpers from
   `@damatjs/framework`.
2. Add `links: "./src/links"` (or an array of paths) to `defineConfig` in
   `damat.config.ts`.
3. Run your migrations and type generation as usual — the link junction tables and
   types flow through the existing pipelines.
4. At runtime, use `getModule("link")` for `create` / `dismiss` / `fetch` / `graph`.
