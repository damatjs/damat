# @damatjs/framework 0.6.0

> Auth middleware fails closed: a route that declares an auth type with no
> registered handler now rejects the request with a structured 401 instead of
> warning and letting it through.

## What changed

`createAuthMiddleware` treated a missing auth implementation as a pass-through:
it logged a warning and executed the route anyway. A typo'd or not-yet-wired
auth type silently exposed the route.

- **Before:** unimplemented auth type → warning log → request proceeds.
- **After:** unimplemented auth type → structured `401 UNAUTHORIZED` response
  (`code: "UNAUTHORIZED"`); the route handler never runs.

## Upgrade notes

If any route declares an auth type you never implemented, it stops working —
that route was unprotected. Register the auth handler (or remove the auth
declaration for genuinely public routes).
