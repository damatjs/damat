# @damatjs/types 0.1.0

> First published minor release: the stable baseline of the zero-dependency
> `AppError` HTTP-error hierarchy and the legacy `initFramework` helper.

## What changed

This is the package's "first of its kind" release — it marks the point where the
contents of `src/index.ts` were stabilized for the 0.1.x line. There is no prior
behavior to migrate from; this note introduces what the package provides at the
baseline.

`@damatjs/types` is the lowest-level package in the Damat stack. It has **zero
runtime dependencies** and ships a single `.` entry point (`dist/index.js`), so any
other package can import it without pulling in anything else. Its whole surface is a
single-root error hierarchy plus one legacy no-op.

The error hierarchy is built so that a failure thrown deep in the ORM, services, or
workflow layers carries a numeric `statusCode` and a stable machine-readable `code`
all the way up to the HTTP boundary. A single `instanceof AppError` check at that
boundary can map any of these to a response without knowing the concrete subclass.

```ts
import { AppError, NotFoundError, RateLimitError } from "@damatjs/types";

throw new NotFoundError(`User ${id} not found`); // 404, code "NOT_FOUND"
throw new RateLimitError("Too many requests", { retryAfter: 30 }); // 429

try {
  // ...
} catch (err) {
  if (err instanceof AppError) {
    sendResponse(err.statusCode, {
      code: err.code,
      message: err.message,
      details: err.details,
    });
  } else {
    throw err;
  }
}
```

## Added

- **`AppError`** — base class extending `Error` with `readonly statusCode`,
  `readonly code`, and optional `readonly details: unknown`. Constructor
  `(message, statusCode = 500, code = "INTERNAL_ERROR", details?)`. Sets
  `this.name = "AppError"`.
- **`ValidationError`** — `extends AppError` → 400, code `VALIDATION_ERROR`,
  constructor `(message, details?)`.
- **`AuthenticationError`** — `extends AppError` → 401, code `UNAUTHORIZED`,
  constructor `(message = "Authentication required")`.
- **`AuthorizationError`** — `extends AppError` → 403, code `FORBIDDEN`,
  constructor `(message = "Access denied")`.
- **`NotFoundError`** — `extends AppError` → 404, code `NOT_FOUND`, constructor
  `(message = "Not found")`.
- **`RateLimitError`** — `extends AppError` → 429, code `RATE_LIMITED`, constructor
  `(message, details?: { retryAfter?: number })` — the only subclass with a typed
  `details` shape.
- **`initFramework`** — legacy no-op kept for backwards compatibility: logs
  `"Framework initialized"` and returns `true`. Not part of the error system.

## Action required

None — this is the baseline release. If you are adopting the package, depend on it
via the workspace `"*"` range inside the monorepo, or `bun add @damatjs/types`
externally, and throw the subclasses instead of bare `Error`s so your HTTP boundary
can read `statusCode`/`code`/`details` uniformly.
