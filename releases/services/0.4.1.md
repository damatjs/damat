# @damatjs/services 0.4.1

> `resolveModel` falls back to a table-name lookup, matching the same fix in
> `@damatjs/orm-pg` — cross-module cascade deletes now resolve snake_case child
> tables.

## What changed

`ModelMethods.resolveModel(name)` resolved a model only by its registry key.
Cascade delete hands it **table names**, so a snake_case child table whose name
differs from its camelCase accessor key failed to resolve mid-cascade.

- **Before:** `registry?.get(name)?.model` — accessor-key lookup only.
- **After:** `registry?.get(name)?.model ?? registry?.getByTableName(name)?.model`.
