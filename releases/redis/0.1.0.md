# @damatjs/redis 0.1.0

> First published minor release: the function-first Redis toolkit for the Damat backend — client lifecycle plus caching, rate limiting, sessions, distributed locks, counters, and a job queue, stabilized for general use.

## What changed

`0.1.0` is the baseline minor release of `@damatjs/redis`. It wraps
[ioredis](https://github.com/redis/ioredis) (via `@damatjs/deps/ioredis`) with a
small, function-first toolkit for the Redis patterns a backend keeps reaching
for. The package owns a process-global Redis singleton so feature helpers
(`cacheGet`, `checkRateLimit`, `acquireLock`, …) can be called with no plumbing,
while still accepting an explicit client when one is needed.

Two design conventions define the surface at this baseline:

- **Singleton-or-explicit client.** `initRedis(config)` constructs a global
  `RedisClient` once at startup; every feature helper resolves its client as
  `client ?? getRedis()`. So `cacheGet("key")` uses the global client and
  `cacheGet("key", myRedis)` overrides it.
- **Trailing `client?` argument.** Each feature function takes `client?: Redis`
  as its **last** parameter — signatures are `(...args, client?)`, never
  `(client, ...args)`.

Everything is exported from the single entry point `@damatjs/redis` (no subpath
exports). The package sits in the `core` layer, is re-exported by
`@damatjs/framework`, and is the foundation `@damatjs/workflow-engine` builds its
workflow locks on.

It is deliberately **not** a full job framework (no worker loop, retry scheduler,
or pub/sub) and **not** a Redlock multi-node implementation — locks are
single-instance.

At this baseline the package provides:

- **Client lifecycle** — `RedisClient` (ioredis wrapper with logging,
  `connect`/`disconnect`/`ping`, `isConnected`); the process-global singleton
  (`initRedis`, `connectRedis`, `getRedis`, `getRedisClient`, `hasRedis`,
  `disconnectRedis`); and a standalone factory (`createRedis`,
  `createRedisConnection`, `createRetryStrategy`, `disconnect`). `getRedis` /
  `connectRedis` throw `RedisNotInitializedError` before `initRedis`.
- **Cache** (`cache:`) — JSON helpers (`cacheSet` with a default 300 s TTL,
  `cacheGet`) and raw-string helpers (`cacheSetRaw`, `cacheGetRaw`), plus
  `cacheDelete` and `cacheDeletePattern`. Misses and malformed JSON resolve to
  `null` rather than throwing.
- **Rate limiting** (`ratelimit:`) — a sliding-window limiter backed by a sorted
  set (`checkRateLimit` → `RateLimitResult`) and a multi-window wrapper
  (`checkMultiRateLimit` → `MultiRateLimitResult`).
- **Sessions** (`session:`) — token → JSON-data CRUD with TTL (`setSession`,
  `getSession`, `extendSession`, `deleteSession`) plus a `SessionManager` /
  `createSessionManager` that adds sliding expiration (auto-extend on access).
- **Distributed locks** (`lock:`) — `acquireLock` (`SET NX PX`, returns an owner
  token), Lua-guarded `releaseLock` / `extendLock` (token-checked, so a process
  can never release or extend a lock it no longer owns), `isLocked`, and the
  `withLock` acquire→run→release helper. `LOCK_PREFIX` is exported.
- **Counters** (no prefix) — atomic `incrementCounter` / `decrementCounter`
  (`INCRBY`/`DECRBY`) plus `getCounter`, `setCounter`, `resetCounter`.
- **Job queue** (`queue:<name>:*`) — `RedisQueue`, a priority + delay queue
  backed by sorted sets (`enqueue`/`dequeue`/`updateStatus`/`getJob`/`cancelJob`/
  `getStats`/`clear`), with `PRIORITY_SCORES`. Storage and state-transition
  primitives only — the consume loop and retry policy are the caller's.
- **Errors & types** — `RedisConnectionError`, `RedisNotInitializedError`;
  `RedisConfig` / `RedisClientConfig`, the rate-limit and queue shapes, and the
  `Redis` / `RedisOptions` re-exports from `@damatjs/deps/ioredis`.

## Action required

None — drop-in. New consumers add the dependency and import from the package root:

```bash
bun add @damatjs/redis
```

Initialize the global client once at startup, then call helpers with no client
argument:

```ts
import { initRedis, cacheSet, cacheGet, disconnectRedis } from "@damatjs/redis";

initRedis({ url: process.env.REDIS_URL ?? "redis://localhost:6379" });
await cacheSet("user:123", { name: "Ada" }, 3600);
const user = await cacheGet<{ name: string }>("user:123");
await disconnectRedis();
```
