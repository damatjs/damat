# @damatjs/load-env 0.1.0

> First published minor release: a tiny, zero-dependency `.env` loader that merges an
> environment-aware cascade of files into `process.env` without ever overwriting
> variables that already exist.

## What changed

This is the package's first stable minor release and the baseline for the current API.
`@damatjs/load-env` provides one public function, `loadEnv`, that reads `.env` files from
a working directory and merges them into `process.env`.

`loadEnv(environment, cwd)` builds a four-entry cascade and reads **every** file in it
that exists, in order, lowest priority first:

1. `.env`
2. `.env.local`
3. `.env.{environment}`
4. `.env.{environment}.local`

Keys are layered: a value in a higher-priority file overrides the same key from a
lower-priority file, so the effective precedence is
`.env.{env}.local` > `.env.{env}` > `.env.local` > `.env`. Before the loop, the keys
already present in `process.env` are snapshotted; a parsed key is written only if its
value is truthy **and** it was not in that snapshot â€” so real system/process environment
variables are never clobbered, and empty values (`FOO=`) are skipped.

```ts
import { loadEnv } from "@damatjs/load-env";

// Call once, as early as possible â€” before reading process.env elsewhere.
loadEnv(process.env.NODE_ENV ?? "development", process.cwd());
```

Both arguments are optional: `environment` defaults to `"development"` and `cwd` to
`process.cwd()`. The function returns `void`; its only effect is mutating `process.env`.
Read/parse failures for any candidate are caught, reported via `console.warn`, and
skipped â€” `loadEnv` never throws.

The values are parsed by an internal hand-rolled parser (`parseEnvFile`) that handles
full-line comments (`#`), `KEY=value` pairs, single/double-quoted values, and inline
comments on unquoted values. It does **not** support variable expansion (`${VAR}`),
multiline values, `export ` prefixes, or escape sequences inside quotes. `parseEnvFile`
is used by `loadEnv` and is not re-exported from the package entry point.

## Added
- `loadEnv(environment?, cwd?)` â€” the single public export: read the `.env` cascade in
  `cwd` and merge it into `process.env` (non-overwriting).
- Zero runtime dependencies â€” only Node's built-in `node:fs` and `node:path`.
