# @damatjs/module 0.1.3

> Re-exports the `@damatjs/link` cross-module authoring surface from the single `@damatjs/module` import, and force-closes lingering connections so `startModuleApp().stop()` no longer hangs in dev/test teardown.

## What changed

Two things changed in this package's own code.

### 1. Cross-module link authoring surface

Cross-module links (`@damatjs/link`) — declare a relationship between two models
that live in different modules, get an auto-generated junction table, and
`create` / `dismiss` / `fetch` / `graph` the relationship at runtime via
`getModule("link")` — were wired through the framework and CLI, but a module
author had **no way to reach the link helpers from the single `@damatjs/module`
authoring import**. They had to import them from `@damatjs/framework` directly,
which broke the "one dependency for module authors" rule.

Now `src/authoring.ts` re-exports the link surface, so app code and standalone
module code share the same import:

```ts
// before — link helpers were not on the @damatjs/module surface
import { defineModule, ModuleService } from "@damatjs/module";
import {
  defineLink,
  collectLinkModels,
  defineLinkModule,
} from "@damatjs/framework";
```

```ts
// after — everything from one import
import {
  defineModule,
  ModuleService,
  defineLink,
  collectLinkModels,
  defineLinkModule,
} from "@damatjs/module";
```

The re-export covers `defineLink`, `collectLinkModels`, `defineLinkModule`, and
the types `LinkService`, `LinkDefinition`, `LinkEndpoint`, `LinkOptions`,
`LinkRowRef`, `LinkModelRef` (all sourced from `@damatjs/framework`). The links
themselves still live in the app's `src/links/`; these helpers just give module
authors the same single authoring surface. See
[`@damatjs/link`](../../packages/link/README.md) for what the feature does.

### 2. `startModuleApp().stop()` force-closes connections

A graceful `server.close()` resolves only once **every** connection has ended.
Under Node a keep-alive or still-active socket — for example a POST whose request
body the route never read — never closes on its own, so the dev/test harness hung
forever: `startModuleApp().stop()` inside an `afterAll` would never resolve, and
apps had to add per-route `await c.req.text()` body-drain workarounds to avoid it.

The server-close logic was extracted from `stop()` into an exported `closeServer`
helper that, after calling `close()`, also closes idle and all connections:

```ts
// before (src/runtime/start.ts) — stop() could hang on a lingering socket
stop: async () => {
  await new Promise<void>((resolve, reject) => {
    server.close((err) => (err ? reject(err) : resolve()));
  });
  // …run shutdown handlers
};
```

```ts
// after — closeServer forces lingering connections shut
export async function closeServer(server: ClosableServer): Promise<void> {
  await new Promise<void>((resolve, reject) => {
    server.close((err) => (err ? reject(err) : resolve()));
    server.closeIdleConnections?.();
    server.closeAllConnections?.();
  });
}

stop: async () => {
  await closeServer(server);
  // …run shutdown handlers
};
```

The `closeIdleConnections` / `closeAllConnections` calls are `?.`-guarded, so a
runtime whose server lacks them (e.g. Bun, where `close()` already resolves
promptly) is a harmless no-op. A global request-body-drain middleware was
considered and rejected — it would fully read every otherwise-ignored request
body on the hot path; force-closing connections fixes every route shape at no
per-request cost. A regression guard (`tests/stop.test.ts`) holds a keep-alive
connection open with an unread body and asserts `closeServer` resolves promptly.

## Added

- `defineLink`, `collectLinkModels`, `defineLinkModule` and the `LinkService` /
  `LinkDefinition` / `LinkEndpoint` / `LinkOptions` / `LinkRowRef` / `LinkModelRef`
  types, re-exported from the `@damatjs/module` authoring surface.
- `closeServer(server)` exported from the runtime — closes the HTTP server and
  forces lingering connections shut.

## Changed / improved

- `startModuleApp().stop()` now calls `closeServer`, so teardown no longer hangs
  on keep-alive or unread-body sockets. Apps can drop the per-route
  `await c.req.text()` body-drain workarounds added solely to avoid the hang.

## Breaking

- None. The link re-exports are additive, and the `stop()` change only affects how
  reliably teardown completes.

## Action required

None — drop-in upgrade.

- To use cross-module links from a standalone module, import `defineLink` /
  `collectLinkModels` / `defineLinkModule` from `@damatjs/module` instead of
  `@damatjs/framework` (the framework import keeps working). See the
  [`@damatjs/link` 0.1.3 notes](../link/0.1.3.md) for the full feature.
- If you added per-route `await c.req.text()` calls only to keep
  `startModuleApp().stop()` from hanging, you can remove them.
