# @damatjs/orm-codegen 0.0.2

> First published release: the pure string-generation core that turns a serialized `ModuleSchema` into TypeScript row types, mutation types, enum unions, relation fields, and matching Zod schemas.

## What changed

`0.0.2` is the baseline release of `@damatjs/orm-codegen`. It is a **pure string
factory** — it consumes a `ModuleSchema` (from `@damatjs/orm-type`) and returns
TypeScript source as strings (or a `filename → contents` map). It performs no I/O
and opens no database connection; the caller (the ORM CLI, the module system)
decides where the output goes.

At this baseline the package provides two halves:

**Type mapping (per column)**

- `columnToTsType(col)` — resolves a `ColumnSchema` to a TypeScript type string.
  Enum columns become the named alias (`status_type` → `StatusTypeEnum`); other
  types go through `pgTypeToTsBase`. Arrays wrap to `Array<base>` and nullable
  columns append `| null`, with inline object/union literals parenthesized.
- `columnToZodSchema(col)` — resolves a `ColumnSchema` to a base Zod validator
  string (wrapped in `z.array(...)` for arrays). It does **not** add
  `.optional()`/`.nullable()`; the schema generators apply those.
- `pgTypeToTsBase` / `enumTypeToTsBase` — the exhaustive `ColumnType` → base TS
  switch, targeting what the `pg` driver returns at runtime (`bigint` → `bigint`,
  `bytea` → `Buffer`, `timestamp`/`date` → `Date`, geometric/range types → inline
  object literals).

**Generators (per table / per module)**

- `generateTypes(schema, opts?)` — one combined `.ts` string: enum unions, then per
  table the row `interface`, the `New*` insert type, and the `Update*` partial type.
- `generateZodTypes(schema, opts?)` — one combined `.ts` string of all Zod schemas
  (`new`/`update`/`query`/`id`) for every table, prefixed with the
  `import { z } from "@damatjs/deps/zod";` line.
- `generateTableFile` / `generateZodFile` — a single table's `.ts` / `.zod.ts`,
  with the enum and relation-target imports computed for that file.
- `generateFilesMap(schema, opts?, logger?)` — the file-per-table layout as a
  `Map`: `<table>.ts`, `<table>.zod.ts`, an `enums.ts` (only when the module has
  enums), and an `index.ts` re-exporting them. Filenames map `_` → `-`
  (`order_item` → `order-item.ts`).

**Supporting surface**

- `DEFAULT_AUTO_FIELDS` — the column names omitted from `New*` types (`id`,
  `createdAt`, `created_at`, `updatedAt`, `updated_at`); extendable per call via
  `GenerateTypesOptions.autoFields`.
- `buildRelationMap(relationships)` — group `RelationSchema[]` by source table.
- `relationFields(relations)` — the optional loaded-relation interface fields
  (`belongsTo`/`hasOne` → singular `field?: Target`, `hasMany` → `field?: Target[]`).
- `GenerateTypesOptions` (`autoFields`, `banner`) and `GeneratedFilesMap` types.

## Added
- The full type-mapping and generator surface described above, exported from the
  package root.
- Automatic Zod schema generation alongside the TypeScript types (`new`, `update`,
  `query` with pagination fields, and `id` schemas).
- Relation-aware output: loaded-relation fields on row interfaces and the matching
  cross-file `import type` statements in the file-per-table layout.

## Changed / improved
- None — this is the baseline release.

## Breaking
- None.

## Action required

None — drop-in. New consumers add the dependency and import from the package root:

```bash
bun add @damatjs/orm-codegen
```

```ts
import { generateTypes, generateFilesMap } from "@damatjs/orm-codegen";
```
