# @damatjs/orm-model 0.6.0

> Timestamp columns become real timestamps: `created_at`/`updated_at` are now
> `timestamp with time zone` (they were `date`, silently dropping the time), and
> `updated_at` is non-nullable with a `now()` default.

## What changed

- **Column type:** `created_at` and `updated_at` were generated as `date` —
  the time-of-day component was lost on write. They are now
  `timestamp with time zone`.
- **`updated_at` shape:** previously nullable with no default; now
  `NOT NULL DEFAULT now()`, so every row carries a valid updated timestamp from
  the moment it is inserted.

## Upgrade notes

New tables get the corrected types automatically. Existing tables keep their
`date` columns until you migrate them; to align, generate a migration that
alters the columns, e.g.:

```sql
ALTER TABLE "<table>"
  ALTER COLUMN created_at TYPE timestamptz,
  ALTER COLUMN updated_at TYPE timestamptz,
  ALTER COLUMN updated_at SET DEFAULT now();
UPDATE "<table>" SET updated_at = created_at WHERE updated_at IS NULL;
ALTER TABLE "<table>" ALTER COLUMN updated_at SET NOT NULL;
```
