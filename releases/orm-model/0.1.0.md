# @damatjs/orm-model 0.1.0

> First published minor release: the fluent, type-safe schema-definition DSL for the Damat ORM, stabilized for general use.

## What changed

`0.1.0` is the baseline minor release of `@damatjs/orm-model`. It provides the
layer where you describe your database — tables, columns, enums, indexes,
constraints, and relationships — through a small fluent builder API (`model(...)`

- `columns.*`). A model definition is pure metadata: it produces a serializable
  `TableSchema` (and, grouped, a `ModuleSchema`) that the rest of the stack turns
  into migrations, queries, and generated TypeScript row types. It executes no SQL
  of its own.

At this baseline the package offers:

- **`model(tableName, properties, options?)` + `ModelDefinition`** — define a
  table; fluent `.indexes()` / `.constrain()` / `.timestamps()` / `.softDelete()`
  and `.toTableSchema()` / `.toTsType()`. Timestamps and soft-delete are on by
  default.
- **The `columns` factory** — one method per PostgreSQL column type
  (`id`, `boolean`, `integer` and its `serial`/`bigInt`/… variants, `numeric`,
  `real`, `doublePrecision`, `money`, `text`, `varchar`, `char`, `timestamp`,
  `date`, `time`, `interval`, `json`/`jsonb`, `uuid`, `bytea`, `enum`, `vector`),
  plus relation, index, and constraint builders.
- **`EnumBuilder`** — named PostgreSQL enums (`CREATE TYPE … AS ENUM`).
- **`IndexBuilder` / `ConstraintBuilder`** — indexes and table-level constraints,
  with auto-naming.
- **Relations** — `belongsTo` (owning side, generates the FK column and
  constraint) and `hasMany` / `hasOne` (inverse, metadata only). Targets are
  given as a model reference or a lazy thunk.
- **Relation validation** — `validateRelations` / `assertValidRelations` against
  live builders, and `validateRelationSchemas` / `assertValidRelationSchemas`
  against serialized `RelationSchema[]`, surfacing aggregated
  `RelationValidationError`s.
- **`toModuleSchema(name, models, options?)`** — group models into a
  `ModuleSchema`, hoisting per-table relations into module-level `relationships`.

## Added

- The full column / enum / index / constraint / relation builder surface
  described above.
- Re-export of `@damatjs/orm-type` shapes (`ColumnType`, `TableSchema`,
  `ModuleSchema`, `RelationSchema`, …) from the package root.

## Changed / improved

- None — this is the baseline release.

## Breaking

- None.

## Action required

None — drop-in. New consumers add the dependency and import from the package root:

```bash
bun add @damatjs/orm-model
```
