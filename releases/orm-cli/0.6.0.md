# @damatjs/orm-cli 0.6.0

> `migrate:status` passes the full module descriptor to the migration layer —
> status is tracked by module `name`, discovery by its `resolve` path.

## What changed

The status command handed `@damatjs/orm-migration` a bare resolve path, which
doubled as the tracking key. In 0.6.0 the migration layer keys tracker queries
by module **name** (see [orm-migration 0.6.0](../orm-migration/0.6.0.md)), so the
command now passes the full `OrmModule` descriptor instead.

- **Before:** `migrate:status` → paths only; tracking keyed by path.
- **After:** `migrate:status` → `OrmModule` descriptors; tracking keyed by
  `name`, discovery via `resolve`.

## Upgrade notes

Nothing to do — the command's CLI surface is unchanged.
