# @damatjs/orm-cli 0.1.0

> First minor release of `damat-orm` — the standalone, config-driven CLI for
> Damat ORM migrations and TypeScript type generation.

## What changed

This is the package's baseline release. `@damatjs/orm-cli` ships the `damat-orm`
binary: a thin command layer over the shared `@damatjs/cli` runner and the
`@damatjs/orm-*` packages. It reads a `damat.config.ts` to discover modules and
the database URL, then drives migrations and codegen against each module's
on-disk `models/`, `migrations/`, and `types/` directories.

The CLI exposes two command groups:

- **`migrate`** — `migrate:up` (apply all pending migrations across every
  module), `migrate:status [module]` (applied vs pending counts; also accepts
  `--module`/`-m`), `migrate:list` (modules with migrations + counts), and
  `migrate:create <module>` (initial migration on first run, diff migration
  thereafter; a no-op diff reports "No changes detected." and exits 0).
- **`generate`** — `generate:types <module>` (read a module's models, build a
  schema, and write a file-per-table type map plus `index.ts` into the module's
  `types/` directory).

Config and path handling at this baseline:

- The config file name is fixed to `damat.config.ts`; module `resolve` paths are
  resolved relative to the config file's directory and may be relative or
  absolute.
- The database URL comes from `projectConfig.databaseUrl`, falling back to
  `services.database` (`connectionString`, or host/port/user/password/database
  fields used to build one).
- Per-module subfolders follow a fixed layout: `models/` (input), `migrations/`
  (`.sql` + snapshot), and `types/` (generated output).

## Added

- `damat-orm` binary with `migrate:up`, `migrate:status`, `migrate:list`,
  `migrate:create`, and `generate:types`.
- `loadModules` / `loadDatabaseUrl` / `requireDatabaseUrl` and the `resolve*Path`
  helpers, exported for programmatic/embedding use.

## Changed / improved

- Stabilized core functionality, expanded test coverage, and resolved known
  issues relative to the `0.0.x` pre-alpha line.

## Breaking

- None — first stable minor.

## Action required

None — drop-in for new projects. Add `@damatjs/orm-cli` and run the commands
against a project that has a `damat.config.ts`:

```bash
bun add -d @damatjs/orm-cli@*
bun damat-orm migrate:create user
bun damat-orm migrate:up
bun damat-orm generate:types user
```
