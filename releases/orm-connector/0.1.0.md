# @damatjs/orm-connector 0.1.0

> First published minor release: the standalone PostgreSQL connection/pool manager for the Damat ORM, stabilized for general use.

## What changed

`0.1.0` is the baseline minor release of `@damatjs/orm-connector`. It is the
lowest layer of the Damat ORM stack: it turns a `DbPoolConfigWithExtras` into a
live, observable node-postgres `Pool` and owns that pool's full lifecycle. It
builds no SQL and knows nothing about models or transactions — everything above
it (`@damatjs/orm-pg`, `@damatjs/service`, `@damatjs/framework`) consumes the
plain `Pool` it produces.

At this baseline the package provides:

- **`ConnectionManager`** — the only stateful piece. Wraps a single `pg` `Pool`
  with:
  - `connect()` — **idempotent**; lazily creates the pool, proves connectivity by
    acquiring and releasing one client, and caches the in-flight promise so
    concurrent/repeat callers all receive the same `Pool`.
  - `disconnect()` — ends the pool and resets all internal state (safe to call
    when never connected; reconnection afterwards is supported).
  - `healthCheck()` — runs `SELECT 1`, returns a `ConnectionStatus`, and keeps the
    manager's connected flag in sync with reality.
  - `getPool()` / `getClient()` — hand out the pool or borrow a client (throw
    `ConnectionError` if called before `connect()`).
  - `getPoolStats()` — live `totalCount` / `idleCount` / `waitingCount` (zeros
    when not connected).
  - `isInitialized()` — true only after a verified connection.
- **`ConnectionError`** — an `Error` subclass with a fixed `name` and an optional
  `cause`, wrapping driver errors on connect/disconnect failures and misuse
  (pool accessed before `connect()`).
- **`setupPoolListeners(pool, logger)`** — attaches `error`-level logging to the
  pool's `error` event and `debug`-level logging to `connect` / `acquire` /
  `release` / `remove`. Wired in only when a logger is supplied, so the manager
  is silent by default.
- **`performHealthCheck(pool, updateStatus)`** — runs `SELECT 1`, never throws,
  reports connectivity + pool stats; safe to wire into readiness/liveness probes.
- **`fetchPoolStats(pool | null)`** — pure, synchronous pool-stat reader that
  coalesces missing counters to `0`.
- **`productionPoolConfig` / `developmentPoolConfig` / `testPoolConfig`** — pool
  sizing/timeout presets that spread caller `overrides` last, so any field can be
  overridden.

## Added

- The full `ConnectionManager` lifecycle API and the `tools/*` helpers
  (`setupPoolListeners`, `performHealthCheck`, `fetchPoolStats`,
  `ConnectionError`) described above.
- The three pool-config presets, re-exported from the package root.

## Changed / improved

- None — this is the baseline release.

## Breaking

- None.

## Action required

None — drop-in. New consumers add the dependency and import from the package root:

```bash
bun add @damatjs/orm-connector
```
