# @damatjs/codegen 1.0.7

> Generated CRUD steps now return `new StepResponse(output, compensateInput?)` and use the engine's new compensation signature `(compensateInput, ctx)`.

## What changed

The per-table CRUD slice codegen scaffolds (`workflows/<table>/steps/…`) is
regenerated to match `@damatjs/workflow-engine`'s new step contract: a step's
forward function returns a `StepResponse` carrying its downstream `output` and an
optional `compensateInput`, and the compensation receives `(compensateInput, ctx)`.

The `update` step is the clearest win — it snapshots the previous row and returns
it as the rollback payload, with no side channel:

```ts
// Before — compensation got (input, _updated, ctx); no clean way to see the prior row
export const updateItemsStep = createStep<UpdateItemsInput, Items>(
  "shop.items.update",
  async (input, ctx) => { /* … */ return row; },
  async (input, _updated, ctx) => { /* … */ },
);

// After — the previous row rides on the StepResponse; compensation restores from it
export const updateItemsStep = createStep<UpdateItemsInput, Items, Items>(
  "shop.items.update",
  async (input, _ctx) => {
    const previous = await service.items.find({ where: { id: input.id } });
    if (!previous) throw new Error("items not found");
    const rows = await service.items.update({ where: { id: input.id }, data: input.data });
    return new StepResponse(rows[0], previous);
  },
  async (previous, _ctx) => {
    await service.items.update({ where: { id: previous.id }, data: { ...previous } });
  },
);
```

Per operation:
- **create** → `new StepResponse(created, created)`; compensation deletes by id.
- **update** → `new StepResponse(row, previous)`; compensation restores `previous`.
- **delete** → `new StepResponse(true, existing)` — output is a `boolean`;
  compensation recreates `existing`. The generated `delete` **workflow** now
  returns `boolean`.
- **find / findMany** → `new StepResponse(result)` — output only, no compensation.

## Changed / improved
- Every generated step wraps its return in `StepResponse`; steps with rollback
  carry the payload explicitly via the third `createStep` generic.
- The generated `delete` workflow output type changed from `<row> | null` to
  `boolean`.

## Breaking
- **Regenerated CRUD requires `@damatjs/workflow-engine` with `StepResponse`.**
  Generated steps import and use it, so the app must be on the matching engine
  version.
- **`delete` returns `boolean`.** A regenerated delete workflow/route returns
  `true` on success instead of the deleted row.

## Action required

Scaffolded files are **scaffold-once** — existing generated steps are never
overwritten. To adopt the new shape, delete and re-generate the CRUD slice for a
table, or hand-edit existing steps to return `StepResponse` and use the
`(compensateInput, ctx)` compensation signature. New tables get it automatically.
Bump `@damatjs/workflow-engine` in lockstep.
