# @damatjs/codegen 2.0.3

> Self-referential tables no longer generate a self-import: relation imports
> skip the table's own type, fixing the TS2440 (`import declaration conflicts
with local declaration`) error that broke the type-check gate on tree-shaped
> schemas.

## What changed

File-per-table generation emitted an import for **every** relation target —
including the table itself. A self-referential model (a category tree with
`parent`/`children`, a comment thread, an org hierarchy) produced:

```ts
// category.ts — before
import type { Category } from "./category"; // TS2440: conflicts with the local declaration
export interface Category { … }
```

`getRelationImports` now filters relations whose target table equals the source
table (`r.to !== tableName`); relation fields reference the locally declared
type directly:

```ts
// category.ts — after
export interface Category {
  id: string;
  parent?: Category; // locally scoped — no import needed
  children?: Category[];
}
```

Regression coverage was added at the helper level (`getRelationImports`) and on
the full generated file output for a self-referential schema.

## Upgrade notes

Re-run codegen on any module with self-referential tables; the regenerated files
compile without the TS2440 workaround.
