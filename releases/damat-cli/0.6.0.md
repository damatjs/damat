# @damatjs/damat-cli 0.6.0

> `damat module add` becomes safe by default: non-registry sources and lifecycle
> scripts require explicit opt-in flags, module ids and paths are validated
> before anything is written, and the CLI reports its real version.

## What changed

### Install security gates

- **Source trust:** installing from a git URL or local path now requires
  `--allow-unverified`; registry installs remain the default trusted path.
  Path sources are validated against `..` traversal escapes.
- **Pre-write validation:** the module id and target directory are checked
  (new `guard.ts` helpers) before any file lands in `src/modules/`.
- **Dependency specs:** a module's dependencies are validated before install —
  `file:`/`git+` specs and unsafe ranges are rejected unless
  `--allow-unverified` is set.
- **Lifecycle scripts:** dependencies install with `--ignore-scripts` by
  default; `--allow-scripts` opts in.

### Version reporting

The CLI's version is embedded at build time (`scripts/embedVersion.ts` →
`version.generated.ts`) instead of read at runtime, so `damat --version` is
correct in every install context.

## Upgrade notes

Scripted installs from git URLs or local paths must add `--allow-unverified`
(and `--allow-scripts` if the module's dependencies genuinely need lifecycle
scripts). Registry installs are unchanged.
