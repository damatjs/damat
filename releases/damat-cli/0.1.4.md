# @damatjs/damat-cli 0.1.4

> `damat module init` now emits a root `README.md` and the full `AGENTS.md`
> authoring guide, and the codegen commands scaffold a per-table CRUD slice.

## What changed

### `damat module init` ships the authoring guide

A freshly scaffolded module package now includes a root `README.md` and a complete
`AGENTS.md` — the same authoring guide as the `damat-modules` skill (codegen-first
flow, one-way route → workflow → step → service layering, "no big files"). The
guide is embedded into the CLI as compiled code by `scripts/embedAgents.ts`
(guarded by `prepublishOnly`) so it reliably ships in the published package instead
of depending on a file npm might not include.

### Codegen scaffolds CRUD

`damat codegen` (app) and `damat module codegen` (package) now generate, beyond row
types + zod + the typed `registry.ts`, a **scaffold-once** per-table CRUD slice
(steps, workflows, and `api/routes`). Existing files are never overwritten, so your
edits survive re-runs. The generation itself lives in
[`@damatjs/codegen` 0.1.4](../codegen/0.1.4.md); the CLI wires the app- and
module-mode output roots.

## Added

- `damat module init` output: a root `README.md` + the embedded `AGENTS.md`.
- `scripts/embedAgents.ts` build step (`agents.generated.ts`).

## Changed / improved

- `damat codegen` / `damat module codegen` report the count of scaffolded CRUD
  files alongside the generated type files, and their `--help` descriptions now
  state the registry + CRUD scaffold rather than just "types".

## Breaking

- None.

## Action required

- None — drop-in. Run `damat module init <name>` to get the new `README.md` /
  `AGENTS.md`, and re-run codegen to materialize the CRUD scaffold.
