# @damatjs/pipelines 1.0.0

> Introduces durable, visual-ready orchestration across jobs, events, workflows, waits, branches, and child pipelines.

## What changed

This is the first release of the pipeline package. It adds a PostgreSQL-canonical
graph runtime with Redis acceleration, immutable code/web definition versions,
transactional starts and signals, and headless authoring and inspection clients.

## Added

- Bounded graph definitions for actions, jobs, workflows, event publish/wait,
  signals, delays, conditions, forks, joins, child pipelines, foreach, and loops.
- Explicit graph output mappings for deterministic branched and forked results.
- JSON-schema validation and a registered capability catalog for safe visual
  authoring.
- Explicit JSON-safe action/workflow output guidance, including ISO
  serialization for ORM `Date` values before durable storage.
- Strict closed-AST validation for node kinds, references, expressions, edge
  outcomes, triggers, and JSON values before code sync or web publication.
- PostgreSQL migrations for definitions, versions, drafts, layouts, runs,
  executions, transitions, signals, triggers, controls, and activity.
- Database-enforced ownership for active and run versions, parent executions,
  transitions, consumed signals, and node-scoped activity.
- A durable router and internal fenced job worker using the shared process
  coordinator and acceleration outbox.
- Transaction-owned starts and signals request one coalesced relay flush after
  commit and no wake-up after rollback.
- Run-and-node claim fencing that serializes routing with administrative
  controls across multiple pipeline processes.
- Version-pinned execution, child lineage, compensation, concurrency-bounded
  foreach, interval/UTC cron/event triggers, and buffered external signals.
- Compensation-failure records retain both the initiating failure and the
  compensation error for operational diagnosis.
- Successful compensation preserves the initiating failure instead of replacing
  it with a synthesized successful-job error.
- Web publication assigns a unique immutable source identity, so publishing a
  later draft after deleting an earlier one cannot collide with stored history.
- Repeatable-read inspection executes job and graph queries sequentially on its
  transaction client, avoiding overlapping-query behavior rejected by pg 9.
- Headless authoring, inspection, administration, retention, and identity-only
  invalidation contracts. No HTTP routes or UI are mounted.
- Hidden inspection preserves graph shape and operational state while removing
  value expressions, runtime content, detailed errors, logs, and activity data.

## Fixed

- Storage-backed router tests now wait for the target run or node state instead
  of assuming global router batches contain only work created by the test.
- Inspection signing-key coverage now matches the shared durability contract:
  signing keys must be non-empty strings or byte arrays.

## Breaking

- None. This is a new package.

## Action required

Add `services.pipelines`, register all referenced capabilities before startup,
run `damat-orm migrate:up`, and include `pipelines` in an explicit runtime worker
selection. Direct job nodes require `services.jobs`; event nodes and triggers
require `services.events.durable`. Redis users need channel access covering
`&damat:*` when acceleration is enabled.
