# @damatjs/orm-pg 0.3.0

> Adds an execution path for bulk upsert: `PgRepository.upsertMany` and `PgModelClient.upsertMany`. Additive — nothing existing changes.

## What changed

The `upsertMany` bulk insert-on-conflict SQL was already built by
`ModelAccessor.upsertMany`, but there was no way to execute it — only single-row
`upsert` ran. This release wires the bulk path through the client and repository:

- **`PgRepository.upsertMany(opt)` → `Promise<T[]>`** returns every affected row.
- **`PgModelClient.upsertMany(options)` → `Promise<PgInsertResult<T>>`** executes
  the accessor's bulk `INSERT … ON CONFLICT … DO UPDATE` and returns
  `{ rows, rowCount, descriptor }`, mirroring `upsert`.
- New internal executor `executeUpsertMany` (mirror of `executeUpsert`).

`UpsertManyOptions` (`{ data, onConflict, updateColumns?, set?, returning? }`) is
re-exported from the client types alongside `UpsertOptions`.

## Upgrade notes

Nothing to do. `upsertMany` is a new method; all existing repository and client
methods are unchanged.

`@damatjs/services` 0.3.0 surfaces this method as `ModelMethods.upsertMany`.
