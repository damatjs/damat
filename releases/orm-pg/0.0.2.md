# @damatjs/orm-pg 0.0.2

> First pre-alpha release: the PostgreSQL execution layer for Damat — a four-layer stack that turns a model definition plus a `pg` Pool into parameterised SQL, typed rows, and a JSON query descriptor.

## What changed

`0.0.2` is the package's first published version (pre-alpha, not intended for
production). It establishes the layered SQL-execution core that everything above
the connector builds on. A consumer gives it a model
(`@damatjs/orm-model`) and a node-postgres `Pool`, and it builds the SQL, runs
it, and returns typed results.

The package ships as a single export entry (`@damatjs/orm-pg`) with the public
barrel already in its present shape:

```ts
export * from "./manager";
export * from "./transaction";
export * from "./repository";
export * from "./executor";
export * from "./client";
export type * from "./types";
export const EntityManager = PgEntityManager; // alias
```

At this baseline the layers are:

- **manager/** — `PgEntityManager` (caches one `PgRepository` per registered
  model, runs transactions, runs raw SQL) and `TransactionalEntityManager` (the
  `tx` object with dynamic `tx.<model>` accessors and savepoint methods).
  `PgEntityManager` is also exported as `EntityManager`.
- **repository/** — `PgRepository` (ergonomic CRUD: `findMany` / `findOne` /
  `findById` / `findManyByIds` / `create` / `createMany` / `update` /
  `updateOne` / `delete` / `deleteById` / `upsert` / `count` / `exists`) plus
  the `createRepository` factory.
- **client/** — `PgModelClient`, per-model CRUD returning
  `{ rows, rowCount, descriptor }`, with thin `ops/find`, `ops/mutate`, and
  `ops/transaction` wrappers.
- **query/** — the pure layer: `ModelAccessor` and the five builders
  (`SelectBuilder` / `InsertBuilder` / `UpdateBuilder` / `DeleteBuilder` /
  `UpsertBuilder`) over `QueryBase`, each producing **both** a parameterised
  `BuiltQuery` and a JSON descriptor; plus where/ident/clause/assert helpers and
  the relation resolver that compiles `with: { … }` into `LEFT JOIN LATERAL`
  subqueries.
- **executor/** — `pgExecuteRaw` and `pgTransaction`, which run a `BuiltQuery`
  against a `Pool`/`PoolClient` with query/slow/error logging.
- **transaction/** — `TransactionManager` and `TransactionContext`
  (BEGIN/COMMIT/ROLLBACK and savepoints).

Baseline design invariants already in place: identifiers are always quoted and
values always parameterised; unknown columns/relations fail fast before SQL is
emitted; and mass UPDATE/DELETE without a WHERE clause is rejected unless
explicitly allowed.
