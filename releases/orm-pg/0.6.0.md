# @damatjs/orm-pg 0.6.0

> `ORDER BY` inputs are validated against whitelists before they reach SQL —
> direction and nulls placement can no longer carry arbitrary strings into the
> query text.

## What changed

`buildOrderByClause` interpolated the caller-supplied direction and nulls
strings into the SQL text as-is. Values are now upper-cased and checked against
whitelists before interpolation:

- direction must be `ASC` or `DESC`;
- nulls must be `NULLS FIRST` or `NULLS LAST`;
- anything else throws a structured error, e.g.
  `[query:orderBy] Invalid direction "…"`.

## Upgrade notes

Nothing to do for well-formed queries — `asc`/`desc` in any casing still work.
Code that passed arbitrary strings through `orderBy` now gets an error instead
of raw SQL interpolation.
