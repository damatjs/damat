# @damatjs/create-damat-app 0.1.0

> First stable minor release: a Bun-only `create-damat-app` scaffolder that
> bootstraps a Damat backend project or a standalone module from a single command.

## What changed

This is the package's baseline release of its kind. After a series of pre-alpha
build/CI fixes (0.0.x), 0.1.0 marks the scaffolder as stabilized: core
functionality settled, testing expanded, and known issues resolved.

`create-damat-app <name>` provides the `create-damat-app` binary. It runs an
interactive, animated terminal experience (banner, spinner, "damat Tips" fact
box, `@clack/prompts` prompts) on the shared `@damatjs/cli` runner and supports
two modes:

- **Project** (default) — clones the `damat-starter-default` repo, prepares it,
  and starts the dev server so the developer sees a running app immediately.
- **Module** (`--module`) — clones the `damat-starter-module` repo and prepares
  it; no env, no version pinning, and no dev server.

A **project** run, end to end:

1. Clone the starter (`git clone … --depth 1`), strip `.git`/`.github`, then
   `git init -b main` with a `chore: bootstrap project structure` commit.
2. Rewrite `package.json` — set `name`, set `packageManager: bun@<version>` when
   Bun is detected, and pin every `@damatjs/*` dependency (except `@damatjs/ui`)
   to `--version` (default `latest`).
3. Append default env to `.env` (`FRONTEND_CORS`, `AUTH_CORS`, `REDIS_URL`,
   `JWT_SECRET`, `COOKIE_SECRET`).
4. Remove stray non-Bun lockfiles, then `bun install`.
5. Start `bun run dev`; on terminate, print a success box with the run command
   and a GitHub link.

A **module** run does steps 1, 2 (name + `packageManager` only, no version
pinning), and 4, then prints its success box immediately.

Options at this baseline: `--module`, `--repo-url`/`-r`, `--version`/`-v`,
`--directory-path`/`-d`, and the runner-provided global `--verbose`. Command
aliases are `init` and `new`. Project names are validated (no dots, no existing
directory), slugified, and lowercased; missing/invalid names trigger an
interactive prompt with defaults `damat-backend` / `damat-module`.

## Added

- Stable two-mode scaffolder (`project` and `--module`) on the `@damatjs/cli`
  runner, with the animated spinner/fact-box UX.
- `--repo-url` to scaffold from a custom starter fork.
- `--version` pinning of `@damatjs/*` dependencies (excluding `@damatjs/ui`).
- Bun version gate: the CLI checks the running Bun major up front and exits with
  an install link if it is too old.

## Changed / improved

- Stabilized core functionality and expanded the test suite relative to the
  0.0.x pre-alpha line.

## Breaking

- None. (0.1.0 is the first stable minor; behavior is additive over 0.0.x.)

## Action required

None — drop-in upgrade from 0.0.x. The scaffolder is invoked fresh each time
(`bunx @damatjs/create-damat-app@latest …`), so just use the new version. Bun
(major v1 or newer) is required.
