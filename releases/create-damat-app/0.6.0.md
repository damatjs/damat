# @damatjs/create-damat-app 0.6.0

> Git and shell commands run through `execFile` with argument arrays instead of
> interpolated shell strings, and the scaffolder reports its build-time version.

## What changed

- **No more shell interpolation:** `cloneRepo`, `initializeFreshGit`, and the
  command executor invoke binaries via `execFile(cmd, args[])` — arguments with
  spaces or shell metacharacters are passed as array entries, never joined into
  a shell string (e.g. `git ["clone", "--depth", "1", "--", url, dir]`).
- **Version embedding:** the published version is embedded at build time
  (`scripts/embedVersion.ts` → `version.generated.ts`).

## Upgrade notes

Nothing to do — behavior is identical for well-formed inputs; inputs that would
previously have needed shell quoting now just work.
