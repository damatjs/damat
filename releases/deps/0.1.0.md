# @damatjs/deps 0.1.0

> First published minor release: one place to pin and re-export every external runtime dependency shared across the Damat monorepo, exposed both as per-library subpaths and as a namespaced root entry.

## What changed

This is the package's first-of-its-kind release. `@damatjs/deps` sits at the
bottom of the dependency graph: instead of each Damat package depending on
`hono`, `zod`, `pg`, and so on directly, every package depends on `@damatjs/deps`
(workspace `*`) and imports the libraries through it. That gives the whole
monorepo a single pinned version of each dependency, a single place to upgrade,
and a curated re-export surface. Earlier `0.0.x` tags were pre-alpha
build-and-packaging iterations; `0.1.0` is the first version stabilized for use.

What the package provides at this baseline — one re-export module per `src/*.ts`,
each wired into the `package.json` `exports` map so the deep import resolves in
published builds:

- **`@damatjs/deps` (root, `src/index.ts`) — namespaced re-exports.** Re-exports
  every library under a namespace (`export * as hono`, `export * as zod`,
  `export * as effect`, `export * as pg`, `export * as ioredis`,
  `export * as nanoid`, `export * as uuid`) so the root import works without name
  collisions between libraries. Usage: `import { zod } from "@damatjs/deps"; zod.z.object({...})`.
  The focused subpaths below are preferred over the root.
- **`@damatjs/deps/hono` (`src/hono.ts`) — a curated Hono surface.** The only
  non-trivial re-export: it pulls together the Hono sub-modules the framework
  actually uses into one import. `export * from "hono"` (`Hono`, types `Context`,
  `MiddlewareHandler`, `Next`) plus `hono/utils/http-status`
  (`ContentfulStatusCode`, …), `hono/http-exception` (`HTTPException`),
  `hono/secure-headers` (`secureHeaders`), `hono/timing` (`timing`),
  `hono/cors` (`cors`), and `serve` re-exported from `@hono/node-server`. So a
  consumer can `import { Hono, cors, serve, HTTPException } from "@damatjs/deps/hono"`
  from a single place.
- **`@damatjs/deps/zod` (`src/zod.ts`) — flat re-export plus a `z` alias.**
  `export * from "zod"` for zod v4 flat, and additionally
  `import * as zod from "zod"; export { zod as z }` so `import { z } from "@damatjs/deps/zod"`
  keeps v3-style `z.object(...)` ergonomics working against v4.
- **`@damatjs/deps/effect` (`src/effect.ts`)** — `export * from "effect"` (full
  Effect-TS surface: `Effect`, `Layer`, `pipe`, …).
- **`@damatjs/deps/pg` (`src/pg.ts`)** — `export * from "pg"` (node-postgres:
  `Pool`, `Client`, `PoolClient`, types).
- **`@damatjs/deps/ioredis` (`src/ioredis.ts`)** — `export * from "ioredis"`
  (`Redis`, type `RedisOptions`; consumed by `@damatjs/redis`).
- **`@damatjs/deps/nanoid` (`src/nanoid.ts`)** — `export * from "nanoid"`
  (`nanoid`, `customAlphabet`; used by the framework for request IDs).
- **`@damatjs/deps/uuid` (`src/uuid.ts`)** — `export * from "uuid"` (`v4`, `v7`,
  `validate`); also reachable via the root namespace (`import { uuid } from "@damatjs/deps"`).

Pinned versions at this baseline (`package.json` `dependencies`): `hono`
`^4.12.0`, `@hono/node-server` `^1.13.7`, `zod` `4.3.6` (**exact** pin, no caret,
for validator/error stability), `effect` `^3.19.18`, `pg` `^8.21.0`, `ioredis`
`^5.9.3`, `nanoid` `^5.1.6`, `uuid` `^13.0.0`.

## Added

- The eight re-export entry points: root namespaced `.` plus the `./hono`,
  `./zod`, `./effect`, `./pg`, `./ioredis`, `./nanoid`, and `./uuid` subpaths.
- The curated `./hono` surface (core Hono + http-status, http-exception,
  secure-headers, timing, cors, and `serve` from `@hono/node-server`).
- The `z` alias on `./zod` for zod v3-style ergonomics against v4.
- Pinned `dependencies` for all eight libraries, with `zod` pinned exactly.

## Changed / improved

- Core functionality stabilized relative to the `0.0.x` pre-alpha line.

## Breaking

- None — this is the baseline minor release.

## Action required

None — drop-in for new projects. Add the dependency and import libraries through
the subpaths so you inherit the pinned versions:

```bash
bun add @damatjs/deps
```

```ts
import { Hono, cors, serve } from "@damatjs/deps/hono";
import { z } from "@damatjs/deps/zod";
```

Inside the monorepo it is referenced via the workspace protocol
(`"@damatjs/deps": "*"`), so workspace packages always resolve the local copy.
