# @damatjs/orm 0.1.0

> First published minor release: the whole Damat ORM behind a single dependency
> and a stable set of subpath exports.

## What changed

`0.1.0` is the package's baseline minor release. `@damatjs/orm` is an umbrella /
meta-package — it carries no logic of its own. Every file under `src/` is a
single `export *` line, and `tsc` compiles each into a matching `dist/*.js` +
`.d.ts` that `package.json#exports` maps to a public subpath. The package exists
so apps and modules can pull in the entire ORM under one dependency and one
import name, while the implementation stays split across focused `@damatjs/orm-*`
packages.

At this baseline the package re-exports five sub-packages:

- **`@damatjs/orm`** (root `.`, `src/index.ts`) — `export *` of all five below,
  so any symbol reachable through a subpath is also reachable from the bare
  import.
- **`@damatjs/orm/model`** → `@damatjs/orm-model` — the column/property DSL,
  schema builders, `toModuleSchema`, and model types/utils.
- **`@damatjs/orm/connector`** → `@damatjs/orm-connector` — `ConnectionManager`
  for pooled PostgreSQL connection lifecycle and health checks.
- **`@damatjs/orm/migration`** → `@damatjs/orm-migration` — migration discovery,
  executor, generator, and tracker (`runMigrations`, `createInitialMigration`,
  `createDiffMigration`, …).
- **`@damatjs/orm/processor`** → `@damatjs/orm-processor` — schema diffing, SQL
  generation, and snapshots (`diff`, `sqlGenerator`, `snapshotExist`).
- **`@damatjs/orm/pg`** → `@damatjs/orm-pg` — the runtime query layer:
  `EntityManager` (aliased `PgEntityManager`), the repository pattern,
  transactions, and the query executor/client.

The CLIs (`@damatjs/orm-cli`, `@damatjs/damat-cli`) deliberately depend on the
individual sub-packages rather than this umbrella; the umbrella is for downstream
consumers.

## Added

- The bare `@damatjs/orm` import re-exporting all five ORM sub-packages.
- Five subpath exports — `./model`, `./connector`, `./migration`, `./processor`,
  `./pg` — each re-exporting the matching `@damatjs/orm-<name>` package.

## Changed / improved

- Stabilized core functionality and resolved known issues relative to the
  `0.0.x` pre-alpha line (this version is in lockstep with the sub-packages it
  re-exports).

## Breaking

- None — first stable minor.

## Action required

None — drop-in for new projects. Add the umbrella and import the whole ORM, or
just the slice you need:

```bash
bun add @damatjs/orm
```

```ts
// the root export pulls in model + connector + migration + processor + pg
import {
  ConnectionManager,
  PgEntityManager,
  runMigrations,
} from "@damatjs/orm";

// or import just one slice via a subpath
import { columns, toModuleSchema } from "@damatjs/orm/model";
import { PgEntityManager } from "@damatjs/orm/pg";
```
