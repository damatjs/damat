# @damatjs/mcp 0.6.0

> The `add_module` tool exposes the CLI's new security gates: `allowUnverified`
> and `allowScripts` inputs map to `--allow-unverified` / `--allow-scripts`, and
> the tool description explains the registry verification gate.

## What changed

`damat module add` (0.6.0) refuses non-registry sources and skips dependency
lifecycle scripts unless explicitly allowed. The MCP server surfaces those gates
instead of hiding them:

- `add_module` gains optional boolean inputs **`allowUnverified`** and
  **`allowScripts`**, documented as security gates in the tool schema.
- The handler forwards `--allow-unverified` / `--allow-scripts` only when the
  input is explicitly `true` — `false`/omitted sends nothing, keeping the CLI's
  safe defaults.
- The tool description now states the behavior: registry sources install
  normally; git/path sources are refused without the opt-in; scripts are skipped
  by default.

## Upgrade notes

AI assistants installing from git URLs or local paths must pass
`allowUnverified: true` (and `allowScripts: true` only when genuinely needed).
Registry installs are unchanged.
