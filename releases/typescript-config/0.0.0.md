# @damatjs/typescript-config 0.0.0

> Baseline: the shared TypeScript compiler presets every Damat package extends.

## What changed

This is the package's baseline. It is a private, never-published package
(`"private": true`, version pinned at `0.0.0`) consumed only through the
workspace as a `devDependency`. It provides three `tsconfig` presets that other
packages extend by relative subpath (e.g.
`"extends": "@damatjs/typescript-config/base.json"`). There are no runtime
exports.

### `base.json` — the shared base

Every backend/library package extends this. Key `compilerOptions` and why:

- **`target` / `lib`: `ES2023`**, **`module`: `ESNext`**,
  **`moduleResolution`: `bundler`** — the monorepo ships pure ESM and runs on
  modern runtimes (Bun / current Node); no down-leveling, and `bundler`
  resolution allows extensionless imports while honouring `exports` maps.
- **`moduleDetection: "force"`** + **`isolatedModules: true`** — every file is
  a module and can be transpiled standalone (required for fast per-file tooling
  and Bun).
- **Full strict mode** — `strict`, `noUncheckedIndexedAccess`,
  `exactOptionalPropertyTypes`, `noImplicitReturns`, `noUnusedLocals`. Maximum
  type safety across the monorepo from one place.
- **`composite: true`** + **`incremental: true`** — enables TypeScript project
  references and `.tsbuildinfo` caching, the foundation for `tsc --build` and
  Turborepo incremental builds.
- **`declaration` + `declarationMap` + `sourceMap`** — emit `.d.ts`,
  declaration maps, and source maps so published types and go-to-definition
  work.
- **`emitDecoratorMetadata` + `experimentalDecorators`** — decorator syntax and
  runtime metadata, used by ORM/model code.
- **`esModuleInterop` + `resolveJsonModule` + `skipLibCheck` +
  `forceConsistentCasingInFileNames`** — smooth CJS/JSON interop, fast builds,
  cross-platform safety.
- **`outDir: "${configDir}/dist"`** with `include`/`exclude` also keyed on
  `${configDir}` — each extending package emits to its own `dist/` and keeps its
  build output, dependencies, and test/mock/fixture/snapshot files out of the
  build, without re-declaring paths.
- **`baseUrl: "../"` + `paths`** — `@damatjs/*` aliases resolve to package
  `src/` during development so the editor and `tsc` follow types before a build
  exists. At runtime packages resolve each other through their real `exports`
  maps, not these aliases.

`base.json` deliberately does **not** set `"types": ["bun"]`; backend packages
add it themselves so the base stays usable by frontend packages.

### `nextjs.json` — Next.js apps

Extends `base.json`; adds the official `next` TS plugin, `jsx: "preserve"`
(Next.js transforms JSX), `allowJs`, `noEmit: true` (Next.js owns the build),
and frontend-oriented `paths` (`@damatjs/ui`, `@damatjs/eslint-config`, ...).

### `react-library.json` — React component libraries

Extends `base.json` and only switches `jsx` to the automatic runtime
(`jsx: "react-jsx"`), so components do not need to import `React` explicitly.

- None.

## Action required

None — this package is internal to the monorepo and is never installed
directly. New workspace packages adopt it by extending a preset, e.g.:

```jsonc
// packages/my-package/tsconfig.json
{
  "extends": "@damatjs/typescript-config/base.json",
  "compilerOptions": { "types": ["bun"], "rootDir": "src", "outDir": "dist" },
  "include": ["src"]
}
```
